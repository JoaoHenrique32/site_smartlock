Executando o Sistema

Este é um guia prático para desenvolvedores que desejam replicar o ambiente do SmartLock e rodar o código-fonte em suas próprias máquinas.

Guia de Execução

Ponto de Partida: Clonar o Repositório

O primeiro passo é obter os arquivos do projeto. Abra o terminal (no Linux/Mac) ou o prompt de comando/PowerShell (no Windows) e execute:

git clone https://github.com/JoaoHenrique32/sistema_smartlock

Isso criará uma pasta com todo o código do sistema e os arquivos de configuração.

Ambiente e Dependências

Para o Backend (Django):

  • Tenha o Python 3.x instalado.
  • Crie um ambiente virtual (opcional, mas recomendado): python -m venv venv e ative-o.
  • Instale as bibliotecas necessárias com pip:
    • pip install requirements.txt

Para o Hardware (NodeMCU):

  • Tenha a Arduino IDE ou o **VS Code com PlatformIO** instalados.
  • No gerenciador de bibliotecas, instale: ESP8266WiFi, PubSubClient (por Nick O'Leary), e ArduinoJson (por Benoit Blanchon).

Configuração do Banco de Dados e MQTT

O sistema depende de um banco de dados **MySQL** e um broker **EMQX**. A forma mais fácil de subir esses serviços é utilizando **Docker**.

Verifique o arquivo settings.py do Django e certifique-se de que as credenciais do banco de dados (usuário, senha, banco) estejam corretas. No código do NodeMCU, configure o endereço IP do broker MQTT.

Rodando o Código

Iniciando o Servidor Web (Django):

python manage.py runserver

Isso fará com que a plataforma fique acessível no seu navegador.

Carregando o Hardware:

Conecte o NodeMCU ao computador via USB. Abra o projeto na sua IDE, selecione a placa **NodeMCU 1.0 (ESP-12E Module)** e a porta COM correspondente, e faça o upload do código.
